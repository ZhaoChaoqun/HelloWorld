# Phase 0: TRIAGE（分诊）

## 目的
快速判断任务复杂度，决定从哪个 Phase 开始执行。

## 由谁执行
Conductor 自己判断，不需要 spawn subagent。

## 判断标准

### TRIVIAL（→ 跳到 Phase 3）
满足以下**全部**条件：
- 任务描述少于 2 句话
- 明确指向单个文件/单个函数的修改
- 不涉及架构变化
- 不涉及多文件联动
- 不涉及新增 API 或数据模型

**示例**：
- "把按钮文案从'确定'改成'提交'"
- "修复 login.py 第 42 行的 typo"
- "给 User 模型加一个 email 字段"

### SMALL（→ 跳到 Phase 2）
满足以下条件：
- 任务目标清晰，不需要讨论方向
- 可能涉及多个文件，但变更模式明确
- 不涉及架构决策或技术选型
- 不需要先出设计方案（非 UI 任务，或 UI 变更很小）

**示例**：
- "添加用户头像上传功能（API 已有，只写前端）"
- "把列表页的分页从前端分页改为后端分页"
- "添加密码强度校验"

### COMPLEX（→ 进入 Phase 1）
满足以下**任一**条件：
- 任务描述模糊或有多种理解方式
- 涉及架构决策（如数据库 schema 设计、API 设计）
- 涉及技术选型（如状态管理方案、认证方案）
- 涉及用户体验取舍（需要讨论交互方案）
- 涉及 UI/前端新页面或新组件设计
- 需要多个角色的视角碰撞才能确定方向

**示例**：
- "实现用户登录功能"
- "加个深色模式"
- "重构数据库访问层"
- "实现实时通知系统"

## 输出格式

```
## TRIAGE 结果

**任务**：[复述任务]
**复杂度**：TRIVIAL / SMALL / COMPLEX
**理由**：[1-2 句话说明判断依据]
**下一步**：进入 Phase X

[如果 COMPLEX 且涉及 UI]
**注意**：涉及 UI，Phase 1 需要先 spawn rams-designer 出设计方案

[如果 TRIVIAL]
**直接执行**：spawn torvalds-coder 执行 [具体任务描述]
```
